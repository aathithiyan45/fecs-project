BASE STATION MANAGEMENT MODULE - COMPLETE WORKING DOCUMENTATION
================================================================

PURPOSE
-------
Provides admin visibility into which user modules are assigned to which base stations, based on the operator who registered them. This creates a hierarchical organizational structure for better monitoring and coordination.


CORE CONCEPT
------------
The module uses an existing relationship chain without requiring new database fields:

User Module (DEV_001)
  ↓ registered_by_emp_id
Operator (EMP001)
  ↓ assigned_station
Base Station (Station-A)

This means:
- When operator EMP001 registers device DEV_001
- And EMP001 is assigned to Station-A
- Then DEV_001 is implicitly assigned to Station-A


COMPLETE WORKFLOW
-----------------

Step 1: Operator Registers Device
----------------------------------
Operator with employee_id EMP001 and assigned_station "Station-A" registers a user module:

Device Registration:
- device_id: DEV_001
- name: Ranger Kumar
- phone_number: 9876543210
- registered_by_emp_id: EMP001 ← Links to operator

Database stores this in device_registrations table.


Step 2: System Creates Implicit Link
-------------------------------------
The system doesn't store station directly in device_registrations.
Instead, it uses a JOIN query to find the station:

device_registrations.registered_by_emp_id = EMP001
  ↓ JOIN
users.employee_id = EMP001
users.assigned_station = "Station-A"

Result: DEV_001 is linked to Station-A through EMP001


Step 3: Admin Views Base Station Management
--------------------------------------------
Admin logs in and clicks "BASE STATIONS" card in dashboard.

The system:
1. Queries all active device registrations
2. JOINs with users table on employee_id
3. Groups devices by assigned_station
4. Displays organized view

SQL QUERY LOGIC
---------------

The core query that powers this module:

SELECT 
  dr.device_id,
  dr.name,
  dr.phone_number,
  dr.registered_by_emp_id,
  dr.registered_at,
  u.assigned_station
FROM device_registrations dr
JOIN users u ON dr.registered_by_emp_id = u.employee_id
WHERE dr.is_active = 1
ORDER BY u.assigned_station, dr.name

This query:
1. Gets all active device registrations
2. Joins with users table on employee_id
3. Retrieves assigned_station from users
4. Orders by station then name
5. Backend groups results by station

EDGE CASES HANDLED
------------------

1. Operator Without Station
   - Shows under "Unassigned" group
   - Still displays device info
   - Admin can identify and fix

2. No Devices Registered
   - Shows empty state message
   - Prompts to register devices
   - Clean UI feedback

3. Multiple Operators Same Station
   - All devices grouped together
   - Shows which operator registered each
   - Clear accountability maintained

4. Operator Changes Station
   - Devices automatically move to new station
   - Historical data preserved
   - No manual updates needed


BUSINESS RULES
--------------

1. Only active devices shown (is_active = 1)
2. Only admin can view this module
3. Grouping based on operator's current assigned_station
4. Devices without valid operator shown as "Unassigned"
5. Station names case-sensitive (Station-A ≠ station-a)


CONCLUSION
----------

The Base Station Management Module provides hierarchical visibility into device organization without requiring additional database fields. It leverages the existing relationship between devices, operators, and stations to create an intuitive admin view.

Key Innovation: Using registered_by_emp_id as the link between devices and stations, creating an implicit but powerful organizational structure.

Result: Admins can see at a glance which base stations have which devices, enabling better resource planning, operational monitoring, and emergency coordination.
